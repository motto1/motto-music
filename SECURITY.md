| ------- | ------------------ |

## 报告安全漏洞

我们非常重视 Motto Music 的安全性。如果你发现了安全漏洞，请**不要**通过公开的 Issue 报告。

### 如何报告

请通过以下方式之一私下报告安全问题：

1. **GitHub Security Advisories**（推荐）
   - 访问 https://github.com/motto1/motto-music/security/advisories
   - 点击 "Report a vulnerability"
   - 填写详细信息

2. **电子邮件**
   - 发送邮件至项目维护者（可在 GitHub Profile 中查找）
   - 邮件标题：`[SECURITY] Motto Music 安全漏洞报告`

### 报告内容应包括

- 漏洞的详细描述
- 受影响的版本
- 复现步骤（越详细越好）
- 潜在的影响和风险评估
- 如有可能，提供修复建议

### 响应流程

1. **确认**：我们会在 48 小时内确认收到你的报告
2. **评估**：我们会评估漏洞的严重性和影响范围
3. **修复**：我们会尽快开发补丁
4. **发布**：修复完成后，我们会发布安全更新并公开致谢报告者（如果你愿意）

### 披露政策

- 请给我们合理的时间来修复漏洞（通常 90 天）
- 在我们发布修复之前，请不要公开披露漏洞
- 我们会在安全更新中公开致谢报告者（除非你希望匿名）

## 安全最佳实践

### 用户

- 始终使用最新版本的 Motto Music
- 不要在不受信任的网络环境下使用 Bilibili 登录功能
- 定期检查应用权限设置
- 注意保护你的 Bilibili 账号凭证

### 开发者

- 遵循 [CONTRIBUTING.md](CONTRIBUTING.md) 中的代码规范
- 不要在代码中硬编码敏感信息（API 密钥、密码等）
- 使用环境变量或安全存储方式管理凭证
- 对用户输入进行适当的验证和清理
- 保持依赖项更新，及时修复已知漏洞

## 已知安全考虑

### Bilibili API 集成

- 用户凭证使用平台安全存储（Android KeyStore / iOS Keychain）
- 网络请求遵循 Bilibili 官方 API 规范
- 不会收集或上传用户隐私数据

### 本地存储

- 数据库文件存储在应用沙盒中
- 缓存文件定期清理
- 下载文件保存在用户指定目录

### 第三方依赖

- 我们定期审查和更新依赖项
- 使用 `flutter pub outdated` 检查过期依赖
- 优先使用官方维护的软件包
